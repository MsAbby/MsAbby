hexo更新博客

1. 创建或更新博客并上传

1.1 检查工具的安装

  • 是否注册gitHub、安装node、npm、git Bash、配置SSH
  • 是否安装Hexo

1.1.1 配置SSH

  • 1、打开git bash 面板
  • 2、关联githup账户：
    $ git config - -global user.username "githup用户名"
  • 3、关联邮箱：
    $ git config - -global user.email "your_email@example.com" 再三个回车
  • 4、检查本地是否有SSH Key(公钥)存在：
    $ cd ~/.ssh
  • 5、生成新的SSH Key：
    $ ssh-keygen -t rsa -C "your_email@example.com" 后面直接按回车
  • 6、打开id_rsa.pub： 在.ssh文件夹，找密钥id_rsa.pub文件， 复制key
  • 7、复制ssh key：点githup头像, Settings -> SSH and GPG keys -> New SSH key中, 粘贴即可
  • 8、检查连接： ssh -T git@github.com

1.1.2 安装hexo

  • 1、创建新的文件夹“blog”, 打开“命令提示行” cd 到指定文件夹
  • 2、安装全局hexo：
    D:\blog> cnpm install hexo-cli -g
  • 3、查看版本号：
    D:\blog> hexo -v
  • 4、初始化博客(必须是空文件)：
    D:\blog> hexo init
  • 5、安装局部hexo：
    D:\blog> cnpm install hexo --save
  • 6、生成静态文件：
    D:\blog> hexo g
  • 7、启动服务器：
    D:\blog> hexo server
  • 8、本地预览: loacalhost: 4000
  • 安装报错解决

  • hexo g

    Usage: hexo command

    .

    .

    .

    For more help, you can use 'hexo help [command]' for the detailed information or you can check the docs: http://hexo.io/docs/
  • ● method1 ：cnpm uninstall hexo-cli -g
  • ● method2 ：手动删除文件夹所有信息
  • ● method3 ：重新执行-安装 - 初始化 - 生成 - 启动

1.2 结合博客UI构建更新博客

  • git clone博客UI主题并优化相关配置
  • 创建文章并发布

1.2.1 修改配置上线

  • 1、选择博客UI并下载：Hexo的官网主题专栏 / NexT主题
    $ git clone 主题github地址'
  • 2、修改站点配置文件：（若不借助博客ui,直接在初始化的blog文件中操作）

    ● 文件名区别：

    站点配置文件：文件根目录下的_config.yml

    主题配置文件：主题目录下的_config.yml

    server:

    port: 5000

    compress: true

    header: true

    server:

    language: zh-Hans

    theme: huxblog

    deploy:

    type: git

    repo: 发布的https的git地址 或者 ssh的git地址

    branch: master

  • 3、修改主题配置文件：

    menu:

    home: / || home

    categories: /categories

    archives: /archives

    tags: /tags

    about: /about

    commonweal: /404

  • 4、git部署插件：
    npm install hexo-deployer-git --save
  • 5、发布：

    a、清除缓存： hexo clean

    b、静态文件： hexo generate （简写：hexo g ）

    c、启服务器： hexo server （简写：hexo s ）

    d、部署文件： hexo deploy （简写：hexo d）

  • 发布报错解决

  • Usage: hexo command

    commands

    clean Remove...

    config Get or Set ...

  • ● method1 ：cnpm install hexo-server --save再执行 hexo generate

  • hexo 部署到 github 上，css、js 文件加载不出来

  • ● method1 ：站点配置文件的 root 后面写你 github 仓库的名字，前后斜杠都不能省！

    root: /github 仓库的名字/

  • 6、组合命令发布：

    a、清除缓存： hexo clean

    b、生成并本地预览：hexo s -g

    c、生成并上传： hexo d -g

    d、打开： name.github.io

    $ hexo new page 'categories'

    $ hexo new page 'archives'

    $ hexo new page 'about'

  • 2、新建文章
    $ hexo new 'my-first-blog'

    $ hexo new pageName'
