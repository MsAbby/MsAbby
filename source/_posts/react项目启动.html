优化-场列表性能优化

  • 1. 定义： 大型列表和表哥数据， 会导致页面卡顿， 大量dom节点会重绘重排
  • 2. 优化方案： 懒渲染和可视区域渲染
  • 3. 懒渲染：
    a. 用于场景： 移动端
    b. 原理： 每次只渲染一部分， 等渲染的数据将滚动完时，再渲染下面的部分
    c. 优点：速度快（1000条左右的数据）
    d. 缺点：数据量特大时，页面依旧存在大量dom, 占的内存过多，导致页面卡顿
  • 4. 可视区域渲染：
    a. 用于场景： 一次性展示大量数据（微博、聊天等）
    b. 原理： 只渲染页面可视区域的列表项， 非可视区域数据“不渲染”（额外加载一点数据）， 滚动时动态更新
  • 5. 可视区域渲染：react-virtualized: 高效渲染大量数据
    a. 安装： yarn add react-virtualized
    b. 入口导入： import ' react-virtualized/style.css'
    c. list组件占满屏幕： 导入AutoSizer组件： 拿到暴露出来的height和width, 设置在上面
    d. 右侧索引： 点击高亮， 城市列表滚动-对应的城市高亮
    1. List组件 的 onRowsRendered: 滚动时，拿到页面中渲染行的信息（startIndex）
    2. 点击时： 滚动到对应城市： 调用scrollToRow方法，用于安全滚动到指定位置（必须可见的，出现过就成）
    3. 设置List组件scrollToAlignment 配置项为start， 保证被点击出现在页面顶部
    4. 索引无法正确定位时， 用list组件measureAllRows ，提前计算所有行高度，让scrollToRow方法精确跳转

组件化 - 组件间-样式覆盖的问题-CSS IN JS

  • 1. 原因： 代码执行，加载到同一个项目中， 同名的样式就会相互影响
  • 2. 解决方式1： 手动处理-取不同的类名
  • 3. 解决方式2: CSS IN JS
  • 4. CSS IN JS: 用js 编写css的统称，解决css样式冲突，覆盖的问题
    a. CSS Modules: 推荐-react脚手架已经集成，可以直接使用
    1. 定义对CSS类名重命名（局部作用域）
    2. 实现方式：webpack的css-loader: 类名变成全局唯一的名字
    3. 命名方式：BEM（block, element, modifier）; react中： filename_classname_hash-指定classname就成
    4. 使用：首先-创建[name].module.css； 其次-导入该样式文件； 最后-通过style.样式名，设置样式
    import styles from './index.module.css'; // css module导入
    // 注意： 写法-单个类名设置样式， 不使用嵌套 // 注意： 组件库已有的全局样式： 需要使用 ：global来指定 :global(.am-navbar-title)
    b. style-component
