I. react路由 - 简介

react路由 - 简介

  • 1. 功能： 从一个页面导航到另一个页面
  • 2. 映射规则：url路径和组件的对应关系

II. react路由 - 基本使用

react路由 - 基本使用

  • 1. 安装： yarn add react-router-dom
  • 2. 导入核心组件： Router / Route / Link
    import { BrowerRouter as Router, Route, Link } from 'react-router-dom'
  • 3. Router组件： 包裹整个应用
  • 4. Link 组件：to 路由的入口（路由入口）
  • 5. Route 组件：配置规则 和 要展示的组件（路由出口）

react路由 - Router组件

  • 1. 用法： 包裹整个应用， 一个React应用只需要使用一次
  • 2. 分类：
    HashRouter 和 BrowerRouter
  • 3. HashRouter： 使用url的哈希值实现
  • 4. BrowerRouter：推荐， h5的history api实现

react路由 - Link组件

  • 1. 用法： 用于指定导航链接（a标签）
  • 2. to属性：
    浏览器地址栏中的pathname

react路由 - Route组件

  • 1. 用法： 指定路由展示组件相关信息
  • 2. path属性： 路由规则
  • 3. component属性： 展示的组件

III. react路由 - 执行过程

react路由 - 执行过程

  • 1. 点击link组件， 修改浏览器地址栏中的url
  • 2. react路由监听到地址栏中url 的变化
  • 3. react路由内部遍历所有Router组件, 使用路由规则与pathname 进行匹配

IV. react路由 - 编程式导航

react路由 - 编程式导航

  • 1. 定义： 通过js代码来实现页面跳转
  • 2. 使用：
    this.props.history.push('/')
    this.props.history.go(n)
  • 3. history：react路由提供的， 用于获取浏览器历史记录的相关信息
  • 4. push(path)： 跳转到某个页面， path 是路径
  • 5. go(n)： 前进或后退到某个页面

react路由 - 默认路由

  • 1. 定义： path=" / "
  • 2. 模糊匹配模式： 默认情况下 ， react路由是模糊匹配模式， 所以 / 下面所有pathname都会被匹配
  • 3. 精确匹配模式： 给Route 组件添加 exact 属性， 就变成了精确匹配， 当path和pathname 完全匹配时才会展示
